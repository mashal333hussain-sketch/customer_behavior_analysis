📊 Customer Behavior Analysis Project
🔹 Overview

This project focuses on analyzing customer shopping behavior using data analytics techniques. The workflow includes data loading, cleaning, exploratory data analysis (EDA), SQL-based data storage and querying, and visualization through a Power BI dashboard. The goal is to extract meaningful insights about customer patterns, preferences, and purchasing trends.

🔹 Dataset

The dataset contains customer-related information such as:

Customer ID

Age

Gender

Item Purchased

Category

Purchase Amount

Location

Review Rating

Subscription Status

Shipping Type

This data helps in understanding customer behavior across different segments.

🔹 Tools & Technologies

Python (Pandas, NumPy) → Data loading and cleaning

SQL (PostgreSQL) → Data storage and querying

Jupyter Notebook → Analysis and workflow execution

Power BI → Dashboard and visualization

MS PowerPoint → Final presentation

🔹 Steps Involved
1. Data Loading

Imported dataset using Pandas

Verified structure using .head() and .info()

2. Data Cleaning

Checked for missing values

Ensured correct data types

Removed inconsistencies

3. Exploratory Data Analysis (EDA)

Analyzed customer distribution

Studied purchase patterns

Identified trends by category, gender, and age group

4. Database Integration

Connected Python with PostgreSQL

Created database and tables

Loaded dataset into SQL using to_sql()

5. SQL Queries

Extracted insights such as:

Total customers

Average purchase amount

Category-wise revenue

Customer segmentation

6. Power BI Dashboard

Created an interactive dashboard with:

KPI cards (Total Customers, Avg Rating, Avg Purchase)

Category-wise revenue charts

Subscription analysis (Yes/No)

Age group-based insights

Filters for category, gender, and shipping type

🔹 Dashboard Features

Interactive slicers

Clean UI design

Real-time filtering

Business-focused insights

🔹 Results & Insights

Majority of customers are non-subscribers

Certain categories generate higher revenue

Young adults contribute the most to purchases

Customer behavior varies across gender and region

🔹 How to Run

Open Jupyter Notebook

Run all cells for data loading and analysis

Ensure PostgreSQL is running

Update database credentials in code

Load dataset into PostgreSQL

Open Power BI file (.pbix)

Refresh data and explore dashboard

🔹 Conclusion

This project demonstrates a complete data analytics workflow from raw data to business insights. It highlights how combining Python, SQL, and Power BI can help in making data-driven decisions.
